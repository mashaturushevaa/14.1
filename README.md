# Практична робота 14.1: Edge Computing Gateway

**Виконала:** Марія Турушева  
**Тема:** Локальна обробка даних та автономне прийняття рішень (Edge Processing)

## 1. Архітектура Edge-рішення
Система спроектована як автономний вузол (Edge Node), що забезпечує повний цикл обробки даних без залежності від хмарних сервісів. 



## 2. Стратегія Offline Operations
- **Local Persistence:** Використано MapDB для миттєвого відновлення стану пристроїв (Restore on Startup) без синхронізації з хмарою.
- **Autonomous Decision-Making:** Всі сценарії автоматизації (наприклад, захист від перевантаження) реалізовані через локальний Rules Engine, що мінімізує затримки (latency).
- **Graceful Degradation:** При втраті зв'язку з інтернетом система переходить у спеціальний режим, зберігаючи критичну функціональність керування.

## 3. Технічні характеристики
- **Latency:** < 10мс (локальна обробка).
- **Privacy:** Дані користувача не виходять за межі локальної мережі.
- **Bandwidth Optimization:** Нульовий трафік у хмару під час автономної роботи.

## 4. Результати
Реалізовано повноцінний Edge-шлюз, здатний підтримувати комфорт та безпеку будинку в умовах повної відсутності зовнішнього зв'язку.
